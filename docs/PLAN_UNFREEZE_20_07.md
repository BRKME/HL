## 1. Гистерезис verdict_flip (главная задача)

**Проблема.** Тактика считает WAIT нейтралью (баг 21.06: проход через WAIT
не меняет действия), гвард — поводом для выхода. Итог: 40% выходов
пере-входят быстрее часа, шумовое дёрганье с околонулевым R и утонувшие
алерты (кейс HYPE 15.07).

**Дизайн (согласовать перед кодом, 10 минут):**
- verdict_flip срабатывает, только если вердикт != стороне позиции
  **K подряд прогонов гварда** (K=2 ≈ 4 часа; K — модульная константа).
- Немедленный выход без гистерезиса остаётся для ПРОТИВОПОЛОЖНОГО вердикта
  (LONG-позиция + вердикт SHORT) — это разворот, не шум.
- WAIT-нырок на один прогон → лог «flip pending 1/K», без алерта.
- Состояние счётчика — в guard_state.json рядом с last_regime.

**Данные для калибровки K (снять 20.07 одной командой):**
распределение длительности WAIT-нырков из tactical_journal + verdict_journal.
Если медианный нырок 1 прогон — K=2 достаточно; если 2 — обсудить K=3.

**Учёт эксперимента:** с первого коммита писать в exit-записи журнала
`policy_version: 2` — чтобы пред/пост-выборки не смешивались в статистике.

**Тесты:** нырок 1 прогон → нет выхода; K подряд → выход + алерт;
разворот в SHORT → немедленный выход; счётчик сбрасывается при возврате
вердикта; policy_version в записи.

## 3. Унификация источников вердикта

«⚪ WAIT дн. · 🟢 LONG такт.»). На 20.07 — решение по существу, варианты:

- **A (минимум, рекомендуется):** оставить два источника как есть —

- **B:** гвард и отчёт читают один источник (тактический), дневной

Решение принять ПОСЛЕ п.1: с гистерезисом тактический вердикт перестанет
мигать, и вариант A, вероятно, станет достаточным.
