# KPI-система HL-проекта (зафиксировано 11.06.2026)

Принципы (единые для BRKME): сигналы и реальные деньги меряются раздельно;
каждая цифра с n рядом; ниже порога мощности — «рано судить», не проценты;
слой без измерителя = шум.

| Слой | Вопрос | Журнал | Измеритель | Доставка |
|---|---|---|---|---|
| Тактика (дни) | Сигналы зарабатывают? | `state/tactical_journal.jsonl` (эмитированные + shadow подавленных китами) | first-touch SL vs +2R за 7д → R-мультипли; WR, avg R, срезы по направлению (`tactical_eval`) | вс 09:00 |
| Фильтры | Что стоят правила? | A/B raw-vs-final (`verdict_journal`) + shadow-строки | форвард запрещённого: шорты при BULL, блоки китов | analyze по запросу |
| Суббота (недели) | SKIP медведей бьёт always-DCA? | `decisions.jsonl` | альфа: планнер в рынке только в BUY-недели vs рынок всегда (BTC-прокси, fwd7) | вс 09:00 |
| Деньги | Счёт растёт? | серверная история HL | неделя P&L $ и %, размер счёта | вс 09:00 + ежедневный дайджест |

Пороги честности: тактика — ≥5 закрытых исходов; советник — ≥4 субботы с
созревшим форвардом. Неоднозначная свеча (SL и цель в один день) считается
убытком — консервативно.

Бэкфилл: журнал тактики начат с первых живых сигналов 11.06 02:53 MSK
(LONG BTC @61,544 / LONG ETH @1,623) — KPI меряет реальную историю с нуля.

Изменение порогов/правил оценки — только версионным коммитом с обоснованием,
никогда задним числом под результат.
