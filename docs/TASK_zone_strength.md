# Задача (в проработку): зона-сила как уточнение тактических уровней

**Статус:** кандидат на разработку **после 20 июля 2026** (тактика заморожена
до этой даты; идёт валидация regime edge). Не трогать до разморозки.

**Предусловие — без него не начинать:** из данных валидации определить, где
модель реально ошибается. Активировать ТОЛЬКО если промахи концентрируются
вокруг уровней: SL выбивает на ретестах, входы в отработанные уровни,
развороты ловятся глубоко в зоне. Иначе — закрыть как неактуальную.

*Примечание 08.07: position_guard пишет `exit_reason` в каждый выход —
к 20.07 доля `sl_breach` против `verdict_flip`/`tp_hit` даст прямое
измерение предусловия. Ранний сигнал по n=7–17: промахи направленческие
(шорт-сторона), не уровневые.*

## Идея
Сила supply/demand-зоны определяется глубиной проникновения цены до
разворота. Сильнейший сигнал — фитиль касается края и отскакивает (ордера
не поглощены); слабейший — несколько свечей глубоко внутри (зона выдыхается).
Применяется на РЕТЕСТЕ, не при формировании.

## Этапы проработки (до кода)
1. Формализовать «зону» из свечных данных тактики (compute_indicators,
   swing_lookback=30); связать со swing-уровнями, не плодить сущностей.
2. Метрика глубины проникновения 0..1; пороги «сильно/слабо» калибруются
   на данных, не назначаются.
3. Выбрать ОДНО применение: фильтр входов / коррекция SL / модуляция размера.
4. Бэктест против текущей модели: критерий — измеримое улучшение WR/R.

## Принципы
- TDD: тесты → реализация → бэктест → предпросмотр → коммит.
- Не ломать чистоту валидации — только после вердикта regime-edge.
- Пороги — модульные константы.
- Честная деградация: нет данных для зоны → сигнала нет.
- Добавляем, когда данные показали необходимость, не когда идея красива.

## Критерий успеха
Внедряется тогда и только тогда, когда (а) валидация показала промахи по
уровням И (б) бэктест показал измеримое улучшение. Иначе — закрывается.
